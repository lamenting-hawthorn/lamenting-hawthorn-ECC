# Security Policy

## Supported Use

This repository is a public reference implementation. It is intended for local
development, evaluation, and adaptation into deployment-specific systems.

Before production use, add:

- deployment-specific secrets management
- authenticated infrastructure boundaries
- observability and alerting
- backup and recovery procedures
- database permission and RLS regression coverage

## Reporting a Vulnerability

Do not open a public issue for a suspected security vulnerability.

Instead, report:

- affected component or file
- reproduction steps
- impact assessment
- any suggested mitigation

Until a private reporting channel is established, coordinate disclosure through
the repository owner directly and avoid publishing exploit details before a fix
is available.
